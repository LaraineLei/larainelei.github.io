# 创建问题

创建问题即不通过需求。因为不通过需求时，需要通过创建问题来说明不通过的原因。

详情见需求管理>验收需求

# 修改问题

问题还没有被负责人审核时，需求方可以继续修改问题。其中，问题名称、编码不能修改。可修改如下字段：

  • 语言：译文存在问题的语言，只能选择一种。如果多个语言都存在问题，需要创建多个问题。
  • 期望完成时间：希望问题被项目负责人提交的时间。
  • 问题类型：译文产生问题的原因。
  • 描述：可以对问题进行详细的描述。
  • 附加文件：可以上传文件辅助说明。

# 问题详情

在任何状态下，都能查看问题详情

  • 点击查看详情按钮

  • 问题详情：问题的详细信息。可以下载文件。

  • 问题历史：问题的操作历史。按照操作时间倒序排列。

# 接受问题

接受问题的同时，需要将问题分配给译员。

  • 本地化翻译：翻译途径为本地化部门内部译员进行翻译。需要选择一位可以进行原语言-目标语言双语翻译的译员。

  • 外包翻译：翻译途径为外包商翻译，本地化部门内部译员进行校对。需要选择一家外包商，和一位可以进行原语言-目标语言双语翻译的译员。

# 拒绝问题

  • 拒绝问题时，需要给出拒绝原因。
  • 问题被拒绝后，译员不会收到该问题。
  • 问题被拒绝后，问题的处理流程结束。

# 开始问题

开始问题表示译员开始翻译任务。

  • 开始时间：开始翻译问题时间。默认为当前时间，不可修改。
  • 预计完成时间：提交问题给项目负责人的时间。

开始问题后，问题将会在展示在译员的个人排期日历中的开始时间-预计完成时间这个时间段。问题所属的任务将会在展示在部门排期日历中的开始时间-预计完成时间这个时间段

# 提交问题

译员完成翻译后，上传翻译好的文件，提交问题给项目负责人。

项目负责人先下载译员翻译好的文件，然后将文件上传，提交问题给需求方进行验收。

  • 上传正确格式的译文，提交问题。

# 下载问题

译员提交任务后，可以下载译文。

  • 点击下载按钮，下载文件。

# 验收问题

# 通过问题

  • 点击通过按钮，则问题状态变为结束。

# 不通过问题

不通过问题需要填写不通过的原因。不通过问题后，问题需要再次被负责人进行审核。

  • 描述：用于填写详细原因。
  • 附加文件：可以上传文件辅助说明。
