# 消息管理

消息

由其他角色触发消息发送事件，而产生的消息提醒。

消息触发事件：

功能/角色需求方项目负责人译员
需求提交需求、验收需求审核需求、提交需求/
任务/创建任务提交任务
问题创建问题、验收问题审核问题、提交问题提交问题

# 消息通知

# 消息数量

  • 当有新消息时，导航栏的消息通知图标上，会出现带未处理消息个数的小红点。
  • 清除所有消息后，小红点消失。

# 清除消息

  • 点击清空全部图标，可以在消息通知框中清除所有消息。

  • 鼠标悬停在单条消息上，可以点击清楚图标，可以在消息通知框中清楚这条消息。

# 消息跳转

  • 点击单条消息，可以跳转到消息中提到的需求、任务或问题。

# 消息中心

# 标为已读

  • 选择一条或多条消息，点击标为已读按钮，则消息的状态变为已读。

# 删除消息

  • 选择一条或多条消息，点击删除按钮，确认后，消息永久删除不可撤回。

# 消息跳转

  • 点击单条消息，可以跳转到消息中提到的需求、任务或问题。

# 接受设置

# 消息接收渠道

本地化平台提供了三种消息接收渠道：RTX通知、站内通知、邮件通知。

  • RTX通知：RTX软件上，以本地化平台官方账号进行消息通知。
  • 站内通知：本地化平台内，导航栏的消息通知图标显示消息通知。
  • 邮件通知：发送到用户的公司邮箱进行消息通知。

# 消息接收事件

  • 不同角色的用户，可根据消息触发事件表，设置消息接收事件。
  • 设置后，点击保存按钮即可生效。
