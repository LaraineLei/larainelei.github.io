# 任务管理

任务

由项目负责人分配给译员的双语翻译任务。

需求管理中，不同角色的操作权限：

操作/角色需求方项目负责人译员
创建/✔️/
修改/✔️/
查看/✔️✔️
开始//✔️
提交//✔️
下载/✔️✔️

# 创建任务

创建任务即接受需求。因为接受需求时，要将需求拆分为以语言为单位的任务。

详情见需求管理>审核需求

# 修改任务

在任务未被译员开始翻译时，可对任务进行修改。

  • 本地化翻译：翻译途径为本地化部门内部译员进行翻译。需要选择一位可以进行原语言-目标语言双语翻译的译员。

  • 外包翻译：翻译途径为外包商翻译，本地化部门内部译员进行校对。需要选择一家外包商，和一位可以进行原语言-目标语言双语翻译的译员。

# 任务详情

在任何状态下，都能查看任务详情

  • 点击查看详情按钮

  • 任务详情：任务的详细信息。可以下载文件。

  • 任务历史：需求的操作历史。按照操作时间倒序排列。

# 开始任务

开始任务表示译员开始翻译任务。

  • 开始时间：开始翻译任务时间。默认为当前时间，不可修改。
  • 预计完成时间：提交任务给项目负责人的时间。

开始任务后，任务将会在展示在译员的个人排期日历中的开始时间-预计完成时间这个时间段。

# 提交任务

译员完成翻译后，上传翻译好的文件，提交任务给项目负责人。

  • 根据任务的类型，上传正确格式的译文。

# 下载任务

译员提交任务后，负责人可以下载任务的译文。

注意

译员可重复提交任务，每次重新提交任务后，任务将变为待下载状态。

# 下载

译员提交任务后，可以下载译文。

  • 点击下载按钮，下载译文。

# 合并下载

对属于同一个需求的任务，译员上传译文并提交任务后，项目负责人可以进行合并下载。

  • 点击合并下载按钮
  • 选择要合并的语言，下在合并的译文
